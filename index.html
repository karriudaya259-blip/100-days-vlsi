    Day 6 вҖ“ Latches & Flip-Flops

    Day 6 вҖ“ Latches & FlipвҖ‘Flops: The Memory Cells of Digital Design

    Definition

    The term latch derives from the Old English lГҰccan, meaning "to seize or hold" вҖ“ reflecting its ability to capture and retain a single bit of data. FlipвҖ‘flop is an onomatopoeic term from early computing, mimicking the sound of a circuit toggling between two stable states.

    In digital electronics, a latch is a levelвҖ‘sensitive bistable multivibrator: its output follows the input while an enable signal is active, and latches (holds) the last value when enable is deactivated. A flipвҖ‘flop is an edgeвҖ‘triggered variant: it samples the input only on a clock edge (rising or falling), ignoring input changes at all other times. FlipвҖ‘flops are the fundamental building blocks of synchronous sequential circuits вҖ“ registers, counters, and finite state machines (FSMs).

    2. Core Principles (4 Foundational Pillars)

    • Bistability: CrossвҖ‘coupled NAND or NOR gates create positive feedback. Two stable states (0 or 1) are maintained indefinitely without input. This is the physical basis of memory.
    • Level vs. Edge Sensitivity: Latches are transparent when enable = 1 (or 0). FlipвҖ‘flops are edgeвҖ‘triggered вҖ“ they sample input only on a clock transition. Edge triggering eliminates race conditions and enables synchronous design (all state changes aligned to a global clock).
    • Characteristic & Excitation Tables: The characteristic table defines the next state Q(t+1) given current state Q(t) and inputs. The excitation table is the inverse: given desired Q(t)вҶ’Q(t+1), it shows required inputs. These tables are essential for state machine synthesis and flipвҖ‘flop conversion (e.g., JK to D).
    • Timing Parameters: Setup time (data must be stable before clock edge), hold time (data stable after clock edge), propagation delay (clockвҖ‘toвҖ‘output). Violating these causes metastability вҖ“ an unpredictable, nonвҖ‘digital state.

    3. Mechanism (StepвҖ‘byвҖ‘Step)

    D Latch (LevelвҖ‘Sensitive)

    • Enable = 1: Output Q follows D (transparent).
    • Enable = 0: Output Q holds the last value (latched).

    D FlipвҖ‘Flop (EdgeвҖ‘Triggered)

    • Rising clock edge (вҶ‘): Q = D (sampled).
    • Otherwise: Q unchanged.

    JK FlipвҖ‘Flop

    • J=0, K=0: Hold (Q unchanged).
    • J=0, K=1: Reset (Q=0).
    • J=1, K=0: Set (Q=1).
    • J=1, K=1: Toggle (Q = ~Q).

    T FlipвҖ‘Flop

    • T=0: Hold.
    • T=1: Toggle.

    5. Critical Analysis

    • Pros: Simple, reliable, foundational for sequential logic. FlipвҖ‘flops enable synchronous design, eliminating race conditions.
    • Cons: Latches are transparent вҖ“ if used in a clocked system, they can cause race conditions and unintended latches. Modern synchronous designs use edgeвҖ‘triggered flipвҖ‘flops exclusively for registers.
    • Modern Applications: Registers, counters, frequency dividers, state machines, memory arrays (SRAM uses latches as bit cells).
    • Academic Debates: Latch vs. flipвҖ‘flop in ASIC design вҖ“ some lowвҖ‘power designs use latches, but mainstream digital design favors edgeвҖ‘triggered flipвҖ‘flops.

    6. RealвҖ‘World Uses

    • Registers: Multiple DвҖ‘FFs sharing a common clock (e.g., 8вҖ‘bit register in CPU).
    • Counters: Ripple counters (TвҖ‘FFs), synchronous up/down counters.
    • Frequency Dividers: TвҖ‘FF toggles each clock cycle, dividing frequency by 2.
    • State Machines: Traffic light controllers, sequence detectors.
    • Memory Arrays: SRAM uses crossвҖ‘coupled inverters (similar to latches) as bit cells.

    7. Conclusion & Synthesis

    Latches and flipвҖ‘flops are the atomic memory units of every digital system. Without them, no state retention вҖ“ no processors, no counters, no computers. Understanding their operation, timing, and application is essential for any VLSI or digital design engineer. Mastery of sequential circuits is a prerequisite for advanced topics like pipelines, caches, and microarchitectures.

    A simple JK latch (levelвҖ‘sensitive) oscillates when J=K=1 and clock is high вҖ“ called a race condition. The masterвҖ‘slave configuration solves it:

    • Master stage samples J,K on the positive clock edge.
    • Slave stage outputs the masterвҖҷs state on the negative clock edge.
    • Result: Only one toggle per clock cycle, no oscillation.

    • Binary counters вҖ“ ripple and synchronous counters (e.g., 4вҖ‘bit up counter).
    • Frequency dividers вҖ“ divide clock frequency for digital watches, microcontrollers.
    • Shift registers вҖ“ JK configured as D flipвҖ‘flop (J=K) for data storage.
    • Sequence detectors вҖ“ state machines for pattern recognition (e.g., 101 sequence).
    • Traffic light controllers вҖ“ FSM with JK flipвҖ‘flops for each state.
    • Random number generators вҖ“ Linear Feedback Shift Registers (LFSR) using toggle mode.
    • Debounce circuits вҖ“ JK latch to clean noisy switch inputs.
    • PWM generators вҖ“ JK flipвҖ‘flops create variable duty cycle signals.

    8. Synthesis

    The JK flipвҖ‘flop is the most functionally complete sequential element. It can be configured as any other flipвҖ‘flop (D, T, SR). Master it, and you master counters, dividers, and finite state machines вҖ“ the backbone of digital systems.

    • Connect NAND output to asynchronous clear or synchronous load.

    2. Ring Counter & Johnson Counter

    Ring counter

    A shift register with feedback from the last output to the first input. Only one '1' circulates.

    • N flipвҖ‘flops вҶ’ N states.
    • Example (4вҖ‘bit): 1000 вҶ’ 0100 вҶ’ 0010 вҶ’ 0001 вҶ’ 1000...
    • Applications: State machines, timing generators, control sequencers.

    Johnson counter (twisted ring)

    Feedback from the inverted output of the last flipвҖ‘flop.

    • N flipвҖ‘flops вҶ’ 2N states.
    • Example (3вҖ‘bit): 000 вҶ’ 100 вҶ’ 110 вҶ’ 111 вҶ’ 011 вҶ’ 001 вҶ’ 000...
    • Applications: Decoders, frequency dividers, sequence generators.

    3. SelfвҖ‘Starting Counters

    A counter that reaches the valid sequence from any initial state (including unused states).

    • Prevents lockвҖ‘up in illegal states вҖ“ critical in real hardware.
    • Achieved by designing nextвҖ‘state logic so that all unused states eventually lead to a valid state.
    • RealвҖ‘world importance: PowerвҖ‘up or glitch recovery вҖ“ without selfвҖ‘starting, the circuit may freeze.

    4. Finite State Machines (FSM)

    An FSM has states, inputs, outputs, nextвҖ‘state logic, and output logic. Two common types:

    Output depends onPresent state + inputsPresent state only
    SpeedFaster (output can change without clock)Slower (output changes only on clock edge)
    Number of statesUsually fewerUsually more
    ExampleUART receiver, sequence detectorTraffic light controller, vending machine

    2. Data Types: wire vs reg

    • wire: Continuous assignment (combinational). Cannot store value inside always. Default value: high impedance (z).

    wire a;                // continuous

    • Advantages: wire is efficient for connections; reg gives flexibility.

    3. Continuous Assignment (assign)

    Drives a wire with a combinational expression.

    • Advantages: Simple, always active.
    • Disadvantages: Cannot be used inside always blocks.
    • Use: Combinational logic like adders, muxes, decoders.

    4. Procedural Blocks: always

    Executes statements when an event occurs.

    • Advantages: Models both combinational and sequential logic.
    • Disadvantages: Incorrect sensitivity lists cause simulation-synthesis mismatch.
    • Use: FlipвҖ‘flops, counters, FSMs.

    5. Blocking vs NonвҖ‘blocking Assignments

    TypeOperatorBehaviourUse
    Blocking=Executes in order (like C).Combinational logic inside always @(*)

    6. if-else and case Statements

    Day 12 вҖ“ Metastability & Clock Domain Crossing

    Day 12 вҖ“ Metastability & Clock Domain Crossing (CDC)

    Topic: Reliable data transfer between clock domains.

    This page explains why digital systems sometimes fail randomly and how engineers prevent it вҖ“ essential knowledge for any VLSI or FPGA designer.

    1. What is Metastability?

    A flipвҖ‘flop has two stable states: 0 and 1. If the data input changes too close to the clock edge (violating setup or hold time), the output may enter an unstable, unpredictable state вҖ“ neither 0 nor 1, but an intermediate voltage that eventually settles to one of them after an unbounded delay. This is metastability.

    Clock в”Җв”Җв”җ в”Ңв”Җв”Җв”Җв”Җв”Җв”җ в”Ңв”Җ в”Ӯ в”Ӯ в”Ӯ в”Ӯ Data в”Җв”ҳ в””в”Җв”Җв”Җв”¬в”Җв”ҳ в””в”Җ (change too close to edge) в”Ӯ Output в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҙв”Җв”Җв”Җ metastable region в”Җв”Җв”Җв”Җ settles to 0 or 1

    Consequence: The downstream logic may see different values (some see 0, some see 1), causing system failure, data corruption, or crashes.

    2. When Does Metastability Happen?

    • Asynchronous inputs (e.g., button press, signal from another clock domain)
    • Violation of setup/hold time due to clock skew or data delay
    • Clock domain crossing (CDC) without proper synchronisation

    3. Solution: TwoвҖ‘FlipвҖ‘Flop Synchroniser

    A twoвҖ‘flipвҖ‘flop synchroniser (backвҖ‘toвҖ‘back flipвҖ‘flops) reduces the probability of metastability to an acceptable level (e.g., < 1eвҖ‘12 per year).

    Asynchronous input в”Җв”Җв”¬в”Җв”Җ FF1 в”Җв”Җв”¬в”Җв”Җ FF2 в”Җв”Җ synchronised output в”Ӯ в”Ӯ clock в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ

    How it works:

    • First flipвҖ‘flop (FF1) samples the asynchronous input. It may become metastable.
    • Second flipвҖ‘flop (FF2) samples the output of FF1 after one clock cycle. By then, FF1 has almost certainly settled to a stable value.
    • FF2 output is safe to use in the destination clock domain.
    вҡ пёҸ A twoвҖ‘flipвҖ‘flop synchroniser does not prevent data loss. It only prevents metastable values from propagating.

    4. Clock Domain Crossing (CDC) вҖ“ The Problem

    When data passes from one clock domain (CLK_A) to another (CLK_B) with different frequencies or phases, metastability is almost guaranteed unless proper CDC techniques are used.

    Common CDC scenarios:

    • Fast вҶ’ slow clock (e.g., 200 MHz вҶ’ 50 MHz)
    • Slow вҶ’ fast clock (e.g., 25 MHz вҶ’ 100 MHz)
    • Asynchronous (completely unrelated) clocks

    5. Basic CDC Techniques

    Handshake (request/ack)Slow control signals, multiвҖ‘bitReliable, losslessHigh latency, more logic
    FIFOMultiвҖ‘bit data streams (e.g., memory, UART)Lossless, high throughputLarger area, complex

    6. Why CDC Matters for VLSI

    • Most modern SoCs have multiple clock domains (CPU @ 2GHz, bus @ 500MHz, I2C @ 100kHz, USB @ 480MHz).
    • Improper CDC is the #1 cause of functional bugs in silicon вҖ“ respins cost millions of dollars.
    • GATE EC/CS may not ask deeply, but interviews for VLSI roles always ask CDC basics.

    7. Example: TwoвҖ‘FF Synchroniser Verilog Snippet (Conceptual)

    sync_ff1 <= async_signal;

    The output safe_signal is synchronised and safe to use in the destination clock domain.

    9. Mean Time Between Failures (MTBF) вҖ“ Why It Matters

    MTBF quantifies how often a metastable event propagates to cause system failure. The formula (simplified) is:

    MTBF = e^(t_metastable / П„) / (f_clk * f_data * W)
    • t_metastable : time allowed for metastability to resolve (usually one clock cycle)
    • П„ : flipвҖ‘flop time constant (processвҖ‘dependent)
    • f_clk : sampling clock frequency
    • f_data : input data toggle frequency
    • W : timing window (setup + hold)

    Key takeaway: MTBF improves exponentially with faster flipвҖ‘flops (smaller П„) and slower clocks. A twoвҖ‘FF synchroniser typically achieves MTBF > 1eвҖ‘12 years вҖ“ safe for most designs.

    10. Gray Code for FIFO Pointers вҖ“ Why Adjacent States Differ by 1 Bit

    When transferring multiвҖ‘bit pointers across clock domains (e.g., FIFO read/write pointers), multiple bits could change simultaneously, causing incorrect values if sampled during transition. Gray code solves this by ensuring only one bit changes between consecutive values.

    Binary: 0(000) вҶ’ 1(001) вҶ’ 2(010) вҶ’ 3(011) вҶ’ 4(100) (multiple bits change) Gray: 0(000) вҶ’ 1(001) вҶ’ 2(011) вҶ’ 3(010) вҶ’ 4(110) (only one bit changes)

    Application: FIFO read/write pointers are stored in Gray code before crossing clock domains. After synchronisation, they are converted back to binary for address generation.

    11. Handshake Protocol (Request/Acknowledge) вҖ“ Reliable Transfer for MultiвҖ‘Bit Data

    When data must be transferred losslessly between clock domains, a handshake is used. It works as follows:

    1. Source domain asserts req (request) and places data on the bus.
    2. Destination domain samples req after synchronisation, captures the data, and asserts ack (acknowledge).
    3. Source domain sees synchronised ack and deвҖ‘asserts req.
