Day 3 – Universal Gates

Day 3 – Universal Gates (NAND & NOR)

NAND as Universal Gate

  • NOT = NAND(A, A)
  • AND = NAND(NAND(A,B), NAND(A,B))
  • OR = NAND(NAND(A,A), NAND(B,B))

NOR as Universal Gate

  • NOT = NOR(A, A)
  • OR = NOR(NOR(A,B), NOR(A,B))
  • AND = NOR(NOR(A,A), NOR(B,B))

CMOS Implementation

  • NAND: PMOS parallel, NMOS series → 4 transistors
  • NOR: PMOS series, NMOS parallel → 4 transistors
  • NAND is faster (electrons > holes)

Ripple Carry Adder

4-bit adder using 4 full adders

Propagation delay = 2 × N gate delays

How it works

  • FA0 adds A0 + B0 + 0 → produces S0 and C1
  • FA1 adds A1 + B1 + C1 → produces S1 and C2
  • FA2 adds A2 + B2 + C2 → produces S2 and C3
  • FA3 adds A3 + B3 + C3 → produces S3 and Cout

Propagation Delay

  • Each Full Adder = 2 gate delays (from Cin to Cout)
  • 4-bit RCA delay = 4 × 2 = 8 gate delays
  • 32-bit RCA delay = 64 gate delays (too slow for modern CPUs)

    Day 5 – Carry Look-ahead Adder

    Day 5 – Carry Look-ahead Adder (CLA)

    Why CLA?

    Ripple Carry Adder delay = 2N gate delays. For 32 bits: 64 delays. Too slow.

    CLA delay = constant (4 gate delays for 4 bits).

    Carry Generate (G) and Propagate (P)

    Gi = Ai · Bi

    Pi = Ai ⊕ Bi

    Carry out: Ci+1 = Gi + Pi·Ci

    4-bit CLA Carries

    C1 = G0 + P0·C0
C2 = G1 + P1·G0 + P1·P0·C0
C3 = G2 + P2·G1 + P2·P1·G0 + P2·P1·P0·C0
C4 = G3 + P3·G2 + P3·P2·G1 + P3·P2·P1·G0 + P3·P2·P1·P0·C0

    Delay Calculation

    • Compute Pi, Gi: 1 gate delay
    • Compute carries: 2 gate delays
    • Compute sums: 1 gate delay
    • Total: 4 gate delays

    Day 6 – Latches & Flip-Flops

    Day 6 – Latches & Flip‑Flops: The Memory Cells of Digital Design

    Definition

    The term latch derives from the Old English læccan, meaning "to seize or hold" – reflecting its ability to capture and retain a single bit of data. Flip‑flop is an onomatopoeic term from early computing, mimicking the sound of a circuit toggling between two stable states.

    In digital electronics, a latch is a level‑sensitive bistable multivibrator: its output follows the input while an enable signal is active, and latches (holds) the last value when enable is deactivated. A flip‑flop is an edge‑triggered variant: it samples the input only on a clock edge (rising or falling), ignoring input changes at all other times. Flip‑flops are the fundamental building blocks of synchronous sequential circuits – registers, counters, and finite state machines (FSMs).

    2. Core Principles (4 Foundational Pillars)

    • Bistability: Cross‑coupled NAND or NOR gates create positive feedback. Two stable states (0 or 1) are maintained indefinitely without input. This is the physical basis of memory.
    • Level vs. Edge Sensitivity: Latches are transparent when enable = 1 (or 0). Flip‑flops are edge‑triggered – they sample input only on a clock transition. Edge triggering eliminates race conditions and enables synchronous design (all state changes aligned to a global clock).
    • Characteristic & Excitation Tables: The characteristic table defines the next state Q(t+1) given current state Q(t) and inputs. The excitation table is the inverse: given desired Q(t)→Q(t+1), it shows required inputs. These tables are essential for state machine synthesis and flip‑flop conversion (e.g., JK to D).
    • Timing Parameters: Setup time (data must be stable before clock edge), hold time (data stable after clock edge), propagation delay (clock‑to‑output). Violating these causes metastability – an unpredictable, non‑digital state.

    3. Mechanism (Step‑by‑Step)

    D Latch (Level‑Sensitive)

    • Enable = 1: Output Q follows D (transparent).
    • Enable = 0: Output Q holds the last value (latched).

    D Flip‑Flop (Edge‑Triggered)

    • Rising clock edge (↑): Q = D (sampled).
    • Otherwise: Q unchanged.

    JK Flip‑Flop

    • J=0, K=0: Hold (Q unchanged).
    • J=0, K=1: Reset (Q=0).
    • J=1, K=0: Set (Q=1).
    • J=1, K=1: Toggle (Q = ~Q).

    T Flip‑Flop

    • T=0: Hold.
    • T=1: Toggle.

    5. Critical Analysis

    • Pros: Simple, reliable, foundational for sequential logic. Flip‑flops enable synchronous design, eliminating race conditions.
    • Cons: Latches are transparent – if used in a clocked system, they can cause race conditions and unintended latches. Modern synchronous designs use edge‑triggered flip‑flops exclusively for registers.
    • Modern Applications: Registers, counters, frequency dividers, state machines, memory arrays (SRAM uses latches as bit cells).
    • Academic Debates: Latch vs. flip‑flop in ASIC design – some low‑power designs use latches, but mainstream digital design favors edge‑triggered flip‑flops.

    6. Real‑World Uses

    • Registers: Multiple D‑FFs sharing a common clock (e.g., 8‑bit register in CPU).
    • Counters: Ripple counters (T‑FFs), synchronous up/down counters.
    • Frequency Dividers: T‑FF toggles each clock cycle, dividing frequency by 2.
    • State Machines: Traffic light controllers, sequence detectors.
    • Memory Arrays: SRAM uses cross‑coupled inverters (similar to latches) as bit cells.

    7. Conclusion & Synthesis

    Latches and flip‑flops are the atomic memory units of every digital system. Without them, no state retention – no processors, no counters, no computers. Understanding their operation, timing, and application is essential for any VLSI or digital design engineer. Mastery of sequential circuits is a prerequisite for advanced topics like pipelines, caches, and microarchitectures.
