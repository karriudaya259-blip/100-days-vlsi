Day 11 – Verilog Fundamentals: 15 Essential Topics

Topic: RTL Design with Verilog – Theory, Syntax, and Examples

This page provides a complete reference for the core concepts of Verilog HDL. Each topic includes syntax, code snippets, ASCII diagrams, advantages/disadvantages, and real-world applications. Use this as a study guide before writing your own modules.

1. Module and Port Declaration

What it is: The basic building block. A module encapsulates logic with input/output ports.

┌─────────────────────────────────┐ │ module my_design ( │ │ input wire a, b, │ │ output reg y │ │ ); │ │ // logic inside │ │ endmodule │ └─────────────────────────────────┘ 
module half_adder (
    input a, b,
    output sum, carry
);
    assign sum = a ^ b;
    assign carry = a & b;
endmodule
  • Advantages: Reusable, hierarchical, clear interface.
  • Disadvantages: Cannot have two modules with same name.
  • Use: Every Verilog design starts with a module.

2. Data Types: wire vs reg

  • wire: Continuous assignment (combinational). Cannot store value inside always. Default value: high impedance (z).
  • reg: Can store value. Used inside always blocks. May be combinational or sequential.
wire a;                // continuous
reg b;                 // can be used in always
assign a = x & y;      // wire
always @(*) b = x | y; // reg used combinationally
  • Advantages: wire is efficient for connections; reg gives flexibility.
  • Disadvantages: New users confuse reg with hardware register.
  • Use: wire for module interconnects; reg for sequential or procedural logic.

3. Continuous Assignment (assign)

Drives a wire with a combinational expression.

assign #5 out = in1 & in2; // with 5 time unit delay
  • Advantages: Simple, always active.
  • Disadvantages: Cannot be used inside always blocks.
  • Use: Combinational logic like adders, muxes, decoders.

4. Procedural Blocks: always

Executes statements when an event occurs.

// Sequential (flip-flop)
always @(posedge clk or posedge rst) begin
    if (rst) q <= 0;
    else q <= d;
end

// Combinational
always @(*) begin
    y = a & b;
end
  • Advantages: Models both combinational and sequential logic.
  • Disadvantages: Incorrect sensitivity lists cause simulation-synthesis mismatch.
  • Use: Flip‑flops, counters, FSMs.

5. Blocking vs Non‑blocking Assignments

TypeOperatorBehaviourUse
Blocking=Executes in order (like C).Combinational logic inside always @(*)
Non‑blocking<=All RHS evaluated before any LHS updated.Sequential logic (flip‑flops)
always @(posedge clk) begin
    q1 <= d;      // non‑blocking
    q2 <= q1;     // old q1, not the new one
end
⚠️ Mixing blocking and non‑blocking incorrectly causes race conditions. Rule: use = for combinational, <= for sequential.

6. if-else and case Statements

// if-else (priority logic)
always @(*) begin
    if (sel == 2'b00) out = a;
    else if (sel == 2'b01) out = b;
    else out = c;
end

// case (parallel mux)
always @(*) begin
    case (sel)
        2'b00: out = a;
        2'b01: out = b;
        2'b10: out = c;
        default: out = 0;
    endcase
end
  • Advantages: case is clearer for many conditions.
  • Disadvantages: if-else can create priority logic; case can create parallel mux.
  • Use: Multiplexers, decoders, ALU, FSM next‑state logic.

7. Parameters and Localparam

Compile‑time constants.

module counter #(parameter N = 4) (
    input clk,
    output reg [N-1:0] count
);
    always @(posedge clk) count <= count + 1;
endmodule
  • Advantages: Makes modules reusable (width, depth configurable).
  • Disadvantages: Cannot be changed at runtime.
  • Use: Parameterised modules (FIFO depth, bus width).

8. Generate Blocks

Creates multiple instances or logic conditionally/iteratively at compile time.

generate
    genvar i;
    for (i = 0; i < 4; i++) begin : gen_adder
        full_adder uut (a[i], b[i], c[i], sum[i], c[i+1]);
    end
endgenerate
  • Advantages: Reduces code duplication, creates regular structures.
  • Disadvantages: Can be hard to debug.
  • Use: Ripple carry adders, memory arrays, bus repeaters.

9. Hierarchical Instantiation

Placing one module inside another.

half_adder ha1 (.a(x), .b(y), .sum(s), .carry(c));
full_adder ┌─────────────┐ │ half_adder1 │ │ half_adder2 │ └─────────────┘
  • Advantages: Reuse, clean design.
  • Disadvantages: Deep hierarchies can slow simulation.
  • Use: Building larger circuits from smaller ones.

10. Arrays and Memories

reg [7:0] mem [0:255];   // 256 bytes, each 8 bits
wire [3:0] bus [1:4];    // 4 wires of 4 bits

// Simple RAM model
reg [7:0] ram [0:63];
always @(posedge clk) if (we) ram[addr] <= data_in;
assign data_out = ram[addr];
  • Advantages: Models memories directly.
  • Disadvantages: Synthesis requires specific coding style.
  • Use: Register files, FIFO buffers, look‑up tables.

11. Tasks and Functions

  • Task: Can have delays, multiple outputs.
  • Function: Returns one value, executes in zero time, no delays.
// Function
function [3:0] add4;
    input [3:0] a, b;
    add4 = a + b;
endfunction

// Task (testbench only)
task delay;
    input [7:0] cycles;
    repeat (cycles) @(posedge clk);
endtask
  • Advantages: Encapsulate repeated code.
  • Disadvantages: Functions must be synthesizable (no time).
  • Use: Testbenches (tasks), arithmetic functions.

12. Testbench Basics

A module that applies stimuli and checks responses. No ports.

module tb;
    reg clk, rst;
    wire out;
    dut uut (clk, rst, out);
    always #5 clk = ~clk;
    initial begin
        $dumpfile("dump.vcd");
        $dumpvars;
        rst = 1; #10; rst = 0;
        // stimulus
        $finish;
    end
endmodule
  • Advantages: Verifies design before fabrication.
  • Disadvantages: Writing good testbenches is time‑consuming.
  • Use: Every design must be tested.

13. System Tasks and Compiler Directives

  • $display – print to console
  • $monitor – print when variables change
  • $dumpfile / $dumpvars – create waveform
  • $finish – end simulation
  • $random – generate random numbers
  • `define WIDTH 8 – macro
  • `ifdef ... `endif – conditional compilation
  • `timescale 1ns/1ps – time unit/precision
  • Advantages: Powerful debugging and configuration.
  • Disadvantages: Overuse can make code messy.
  • Use: Testbenches, configuration.

14. Synthesis vs Simulation

AspectSimulationSynthesis
GoalVerify functionalityConvert to gates
TimingIdeal (zero delay or unit)Real (gate delays)
ConstructsAlmost all VerilogSubset (no delays, no initial)
Loopsfor, repeat, while allowedfor allowed if loop count constant
Initial blocksAllowedNot synthesizable (use reset)
⚠️ Write code that works for both simulation and synthesis. Avoid delays, initial blocks, and non‑synthesizable constructs in RTL.

15. FSM Coding Styles

Two‑always block style (recommended):

// Sequential: state register
always @(posedge clk or posedge rst) begin
    if (rst) state <= S0;
    else state <= next_state;
end

// Combinational: next state and output
always @(*) begin
    next_state = state;
    out = 0;
    case (state)
        S0: if (in) next_state = S1;
        S1: if (in) next_state = S2;
        S2: begin
            if (in) next_state = S1;
            out = 1;   // Mealy output
        end
    endcase
end
  • Advantages: Clear, synthesizable, easy to debug.
  • Disadvantages: More code lines than one‑block style.
  • Use: All FSMs in professional designs.
